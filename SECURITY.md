# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not open a public issue**. Public disclosure of security vulnerabilities puts the community at risk.

Instead, report it privately:

**Email:** security@zerverless.com

Please include:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact and severity assessment
- Any suggested fix (optional but appreciated)

---

## What to Expect

| Step | Timeframe |
|------|-----------|
| Acknowledgment of your report | Within 48 hours |
| Initial assessment and severity rating | Within 5 business days |
| Resolution or mitigation plan | Within 30 days (critical issues prioritized) |
| Public disclosure (coordinated) | After fix is released |

We will keep you informed throughout the process and credit you in the security advisory if you wish.

---

## Scope

This policy applies to security vulnerabilities in the source code of this repository. It does not cover:

- Vulnerabilities in third-party dependencies (report those to the respective projects)
- Issues in infrastructure or services not managed in this repo

---

## Supported Versions

We provide security fixes for the latest major version only. We strongly recommend keeping your dependencies up to date.

---

## Responsible Disclosure

We ask that you:

- Give us reasonable time to address the issue before public disclosure
- Avoid accessing, modifying, or deleting data that does not belong to you
- Do not exploit the vulnerability beyond what is necessary to demonstrate it

We commit to not pursuing legal action against researchers who act in good faith and follow this policy.
